# load-balancer-gen-service
# Kubernetes LB Service & Ingress Patcher

Лёгкий Python-контроллер для Kubernetes, который автоматически управляет LoadBalancer сервисами и Ingress ресурсами.

---

## Что делает

Скрипт в реальном времени следит за кластером (через Watch API) и выполняет три задачи:

### 1. Генерация LoadBalancer сервисов

Если сервис содержит аннотацию:

platrorm/external-load-balancing: "N"

Создаётся N копий сервиса:

my-service → my-service-0, my-service-1, ...

Особенности:
- копии получают platrorm/autogenerated=true
- удаляются служебные поля (clusterIP, uid и др.)
- оригинальный сервис помечается и не обрабатывается повторно

---

### 2. Аннотации для Cilium (LB IPAM)

Каждому LoadBalancer сервису добавляется:

lbipam.cilium.io/sharing-key: <namespace>

Это нужно для корректного распределения IP в Cilium.

---

### 3. Патчинг Ingress

Все ingress ресурсы приводятся к:

ingressClassName: nginx

Можно форсировать через переменную:
FORCE_NGINX_INGRESS=True

---

### 4. Watcher и отказоустойчивость

- 2 потока: сервисы и ingress
- автоматический рестарт потоков при падении
- exponential backoff при ошибках

---

## Конфигурация (ENV)

OUT_OF_CLUSTER=True
DEBUG_KUBECONFIG=~/.kube/config_main
DEBUG_CONTEXT=dev
FORCE_NGINX_INGRESS=False
LOGLEVEL=INFO

---

## Запуск

Локально:

export OUT_OF_CLUSTER=True
python3 script.py

Внутри кластера:

export OUT_OF_CLUSTER=False
python3 script.py

---

## RBAC

Требуются права:

- services: get, list, watch, create, patch
- ingresses: get, list, watch, patch

pip install kubernetes

## Коротко

Это простой Kubernetes-контроллер, который:
- масштабирует LoadBalancer сервисы через аннотации
- управляет IP через Cilium
- стандартизирует ingress (nginx)
- работает в реальном времени через Watch API
